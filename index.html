Database Management Project of the Restaurant NAPOIM

Project Overview: Restaurant Management Database System

As part of a team project, we developed a comprehensive database management system for a restaurant named NAPOIM. The project focused on optimizing the organization, management, and analysis of various aspects of restaurant operations, such as orders, employees, dishes, beverages, ingredients, and suppliers. The database tracks essential details about orders, staff (waiters and chefs), and suppliers, enabling the restaurant to improve efficiency in handling transactions and inventory.

My Contributions:

1. Requirement Analysis: I actively contributed to gathering and analyzing the restaurantâ€™s operational needs, ensuring that the database covered essential entities such as orders, employees, and ingredients.

2. Conceptual and Logical Design: I played a key role in designing the Entity-Relationship (E/R) schema and relational model. This involved translating business requirements into a well-structured database, optimizing it for real-time restaurant operations, including the involvement of chefs, waiters, and tracking of orders.

3. Data Volume and Performance Optimization: I worked on calculating and estimating the data volume for a 3-month period, designing the system to handle up to 9,000 orders, 22,500 chef involvements, and 27,000 beverage orders. This allowed the system to handle high-demand periods efficiently.

4. SQL Queries: I implemented several complex SQL queries that were used to generate reports, including:

  • Tracking waiter and chef workloads.
  • Identifying popular dishes and beverages.
  • Monitoring ingredient expiration and restocking needs.
  • Calculating average bills and comparing daily revenues.

Graph Project

GraphProject is an intuitive system designed for generating and visualizing mathematical graphs. Users can input equations and specify range values to visualize their data effectively. This project aims to provide an easy-to-use tool for students, educators, and anyone interested in graphing mathematical concepts.

Features:

  • Equation Input: Users can enter mathematical equations (e.g., y = x2).
  • Graph Visualization: Specify X and Y ranges to generate accurate graphs.
  • User-Friendly Interface: Intuitive design for easy navigation and interaction.
  • API Integration: Scalable API for efficient graph rendering.

Project Structure:

  • Frontend: The user interface for inputting equations and displaying graphs.
  • Backend API: Handles requests for graph generation and supports scalability.
  • CI/CD Pipeline: Automates testing, building, and deployment processes.
